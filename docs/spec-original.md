# 일본어 단어 학습 웹앱 — 기능 정의서 (PRD) v1.0

> 작성: PM/풀스택 관점 정리본 대상 문서: 개발 착수용 기능 명세 + 데이터 모델 + AI 파이프라인 기준 샘플: `N3단어_총정리(와카메)·스터디.pdf` (20p, 약 576단어)

------

## 1. 개요

### 1.1 목적

PDF 단어장(외국어–한국어 뜻)을 업로드하면 AI가 유의어·예문을 붙여주고, 4지선다 퀴즈와 경량 간격반복(SRS)으로 **혼자서 일본어 단어를 효율적으로 외우는** 개인용 학습 도구.

### 1.2 핵심 가치 (차별점)

- 시중 단어앱과 달리 **내가 가진 PDF 단어장을 그대로 학습 콘텐츠로 변환**
- 단순 암기가 아니라 **AI가 유의어·문맥 예문을 붙여줘서 실사용 감각**을 익힘
- **망각곡선 기반 복습**으로 "외웠다고 착각 → 까먹음" 방지

### 1.3 대상 사용자

- 1인 (개인용). 인증은 MVP에서 최소화(무인증 또는 단일 비밀번호).
- DB 스키마에는 `user_id` 컬럼을 예약해 향후 멀티유저 확장 대비.

## 2. 기술 스택 (확정)

| 레이어          | 선택                                 | 이유                                                |
| --------------- | ------------------------------------ | --------------------------------------------------- |
| 프레임워크      | **Next.js (App Router, TypeScript)** | 프론트+백 단일 코드베이스, API Routes로 PDF/AI 처리 |
| DB              | **SQLite + Prisma**                  | 개인용 파일 DB, 운영부담 0, 마이그레이션 간편       |
| AI              | **Claude API** (Messages)            | 단어 파싱 + 유의어·예문 생성                        |
| PDF 텍스트 추출 | **unpdf** (또는 pdf-parse)           | Node 환경, 서버리스 호환                            |
| 스타일          | Tailwind CSS                         | 빠른 UI 구성                                        |
| 배포            | **Vercel** (git push 자동배포)       | WAS·서버세팅 불필요, 무료티어                       |

> 참고: SQLite 파일은 Vercel 서버리스에선 휘발되므로, 배포까지 고려하면 **Turso(SQLite 호환) 또는 Vercel Postgres**로 교체 권장. 로컬 개발/자가호스팅이면 SQLite 파일로 충분.

## 3. 핵심 사용자 플로우

```
[1] PDF 업로드
      ↓  (unpdf 텍스트 추출)
[2] AI 파싱 → 단어 목록 구조화 (漢字/후리가나/뜻)
      ↓  (사용자 검수: 잘못 잡힌 행 수정/삭제)
[3] AI 배치 생성 → 각 단어에 유의어 + 예문(읽기·번역)
      ↓  (진행바: 45/576)
[4] 단어장 완성 → DB 저장
      ↓
[5] 매일: "오늘 복습할 단어" 4지선다 풀이
      ↓
[6] 정답/오답 → SRS 레벨 갱신 → 다음 복습일 재계산
      ↓
[7] 대시보드에서 진도율 확인
```

## 4. 기능 명세

우선순위: **P0 = MVP 필수**, **P1 = MVP 이후 우선**, **P2 = 향후**

### F1. PDF 업로드 & 파싱 (P0)

| 항목      | 내용                                                         |
| --------- | ------------------------------------------------------------ |
| 입력      | `漢字 | 후리가나 | 한국어 뜻` 3단 구조 PDF (한컴 등 다양한 포맷) |
| 처리      | ① unpdf로 layout 모드 텍스트 추출 → ② 텍스트를 청크로 분할해 Claude에 전달 → ③ 단어 배열(JSON) 반환 |
| 파싱 규칙 | **후리가나는 소스값 우선 사용**(대부분 존재), 없는 경우(kana 단어)에만 AI가 읽기 보완. 섹션헤더 `【】`, 예문(☀), 줄바꿈 아티팩트는 AI가 정리 |
| 예외      | 파싱 실패/애매한 행은 별도 표시 → 검수 화면에서 사용자가 수정 |
| 산출      | `word` 레코드 초안 (enrichment 전 상태)                      |

### F2. AI 검수 화면 (P0)

- 파싱 결과를 표로 보여주고 행 단위 **수정·삭제·추가** 가능
- "이 단어장으로 확정" 클릭 시 F3(생성)로 진입
- (PDF 파싱은 앱의 유일한 실패 지점이므로 검수 단계는 필수)

### F3. AI 유의어·예문 생성 (P0)

| 항목      | 내용                                                         |
| --------- | ------------------------------------------------------------ |
| 타이밍    | **업로드 즉시 배치 생성**                                    |
| 호출 방식 | 단어 **15~20개씩 묶어** 1회 호출(JSON 배열 반환) → 호출수·비용 최소화 |
| 생성 내용 | 단어당 **유의어 1~3개** + **예문 2개**(일본어 문장 / 후리가나 읽기 / 한국어 번역) |
| 저장      | 생성 결과 DB 영구 저장 → **재호출 없음**                     |
| UX        | "생성 중… 320/576" 진행바 노출, 실패 청크는 재시도           |

**생성 결과 JSON 스키마(예시):**

```json
{
  "kanji": "暖まる",
  "reading": "あたたまる",
  "meaning_ko": "따뜻해지다, 훈훈해지다",
  "synonyms": ["温まる", "ぬくもる"],
  "examples": [
    {
      "jp": "スープを飲んで体が暖まった。",
      "reading": "スープをのんでからだがあたたまった。",
      "ko": "수프를 마시고 몸이 따뜻해졌다."
    }
  ]
}
```

### F4. 단어장 뷰 (P0)

- 단어 카드 리스트: 漢字 / 후리가나 / 뜻 / 유의어 / 예문
- SRS 레벨 뱃지 표시 (0~5)
- 검색·필터(레벨별, 미학습/복습필요)

### F5. 퀴즈 & 복습 (P0)

| 항목          | 내용                                                         |
| ------------- | ------------------------------------------------------------ |
| 유형          | **4지선다, 양방향**                                          |
| 정방향        | 문제=`漢字/후리가나`, 보기=정답 뜻 + 오답 뜻 3개(같은 단어장 랜덤) |
| 역방향        | 문제=`한국어 뜻`, 보기=정답 단어 + 오답 단어 3개             |
| 출제 우선순위 | ① 오늘 복습 대상(`next_review_date <= today`) → ② 미학습 단어 |
| 결과 처리     | 정답/오답 → F6 SRS 로직으로 레벨·복습일 갱신                 |
| 세션 종료     | 정답률·소요시간·틀린 단어 요약                               |

### F6. 경량 SRS (Leitner 박스) (P0)

- 단어별 **숙련도 레벨 0~5** 보유
- 복습 간격:

| 레벨 | 다음 복습까지     |
| ---- | ----------------- |
| 0    | 당일(즉시 재출제) |
| 1    | +1일              |
| 2    | +3일              |
| 3    | +7일              |
| 4    | +14일             |
| 5    | +30일             |

- **정답**: `level = min(level+1, 5)`, `next_review_date = today + interval[level]`
- **오답**: `level = 0`, `next_review_date = today`(또는 +1일)
- `last_reviewed_at`, `correct_count`, `wrong_count` 기록

### F7. 진도 대시보드 (P0)

- **진도율 = (레벨 4 이상 단어 수) / (전체 단어 수) × 100**
- 오늘 복습할 단어 수, 미학습 단어 수
- 레벨별 분포(0~5) 막대그래프
- 단어장별 진도

### F8. (P1/P2) 확장 기능

- P1: 주관식 입력 퀴즈, 예문 빈칸 채우기 퀴즈
- P1: AI 생성 예문/유의어 **인라인 수정** 기능
- P2: TTS 발음 듣기(일본어 음성 합성)
- P2: 후리가나 맞추기 퀴즈, 반의어·JLPT 레벨·품사
- P2: 멀티유저(로그인), 학습 통계 상세 그래프, 연속 학습(스트릭)

## 5. 데이터 모델 (Prisma 기준 초안)

```prisma
// 단어장
model VocabSet {
  id          Int      @id @default(autoincrement())
  userId      Int?     // 멀티유저 확장 예약
  name        String
  sourceFile  String?
  totalWords  Int      @default(0)
  createdAt   DateTime @default(now())
  words       Word[]
}

// 단어 (파싱 + AI 생성 결과 통합)
model Word {
  id            Int      @id @default(autoincrement())
  setId         Int
  kanji         String?          // 한자 표기 (kana 단어는 null)
  reading       String           // 후리가나 (소스 우선, 없으면 AI)
  meaningKo     String           // 한국어 뜻
  synonyms      String?          // JSON 문자열: ["温まる", ...]
  examples      String?          // JSON 문자열: [{jp, reading, ko}, ...]
  posTag        String?          // 품사 (P2)
  enrichedAt    DateTime?        // AI 생성 완료 시각
  set           VocabSet @relation(fields: [setId], references: [id])
  srs           WordSrs?
  @@index([setId])
}

// SRS 학습 상태
model WordSrs {
  id             Int      @id @default(autoincrement())
  wordId         Int      @unique
  level          Int      @default(0)   // 0~5
  nextReviewDate DateTime @default(now())
  correctCount   Int      @default(0)
  wrongCount     Int      @default(0)
  lastReviewedAt DateTime?
  word           Word     @relation(fields: [wordId], references: [id])
  @@index([nextReviewDate])
}

// 퀴즈 시도 로그 (통계·복습 근거)
model QuizAttempt {
  id         Int      @id @default(autoincrement())
  wordId     Int
  direction  String   // "word_to_meaning" | "meaning_to_word"
  isCorrect  Boolean
  answeredAt DateTime @default(now())
  @@index([wordId])
}
```

## 6. AI 파이프라인 상세

### 6.1 2-Pass 전략 (권장)

파싱과 생성을 분리해 오류 전파를 막는다.

**Pass 1 — 파싱(구조화):**

- 입력: unpdf 추출 텍스트 청크
- 프롬프트: "다음 텍스트에서 일본어 단어를 추출. 각 항목을 `{kanji, reading, meaning_ko}`로. 후리가나 컬럼이 있으면 그 값을 reading으로 사용. 섹션헤더(【】)·예문(☀)·페이지헤더는 제외. JSON 배열만 출력."
- 출력: 검수 화면에 표시

**Pass 2 — 생성(enrichment):**

- 입력: 확정된 단어 15~20개 묶음
- 프롬프트: "각 단어에 유의어 1~3개, 예문 2개(jp/reading/ko) 추가. JSON 배열만 출력."
- 출력: `Word.synonyms`, `Word.examples` 업데이트

### 6.2 비용/속도 최적화

- 576단어 기준 Pass2는 20개 묶음 → **약 29회 호출**
- 청크 실패 시 해당 청크만 재시도(전체 재실행 금지)
- 생성 결과는 DB 저장 → 동일 단어 재호출 없음
- (선택) Pass1·Pass2를 청크당 1회로 합치면 호출 절반, 대신 파싱검증 약해짐 → **정확도 우선이면 2-Pass 유지**

### 6.3 출력 신뢰성

- API 호출 시 시스템 프롬프트에 "JSON 외 텍스트·마크다운 금지" 명시
- 응답 파싱 전 ```json 펜스 제거 후 `JSON.parse`, 실패 시 재시도

## 7. 화면 구성 (MVP)

1. **홈/대시보드** — 진도율, 오늘 복습 수, 단어장 목록, 레벨 분포
2. **업로드** — PDF 드롭존 → 파싱 진행바
3. **검수** — 파싱 결과 표(수정/삭제/추가) → 확정 → 생성 진행바
4. **단어장 상세** — 단어 카드 리스트(후리가나/뜻/유의어/예문/레벨)
5. **복습/퀴즈** — 4지선다 진행 화면
6. **결과** — 세션 요약(정답률·틀린 단어)

## 8. 범위 정의

### MVP (P0)

PDF 업로드 → AI 파싱 → 검수 → AI 유의어·예문 생성 → 단어장 뷰 → 4지선다 양방향 복습 → 경량 SRS → 진도 대시보드

### 제외(향후)

주관식/빈칸 퀴즈, TTS, 반의어·JLPT·품사, 멀티유저, 상세 통계 그래프, 학습 스트릭
