## 요구사항

### R1. PDF 업로드 & 파싱 (F1)

- 사용자는 `漢字 | 후리가나 | 한국어 뜻` 3단 구조 PDF를 **1회 1파일**(최대 10MB) 업로드한다.
- 서버는 unpdf의 layout 모드로 텍스트를 추출하고, 텍스트를 청크로 분할해 **Pass 1**(파싱) 프롬프트로 Gemini 2.5 Flash에 전달한다.
- Gemini는 각 항목을 `{kanji, reading, meaning_ko}` JSON 배열로 반환한다(네이티브 JSON 모드).
  - **후리가나는 소스값 우선.** 소스에 읽기 컬럼이 있으면 그 값을 `reading`으로 사용하고, kana-only 단어(읽기 컬럼 없음)에만 AI가 읽기를 보완한다.
  - 섹션헤더(`【】`), 예문(☀), 페이지헤더, 줄바꿈 아티팩트는 파싱 결과에서 제외한다.
- 파싱 결과는 enrichment 전 상태의 `Word` 레코드 초안으로, 검수 화면에 표시한다.
- 파싱 진행률은 **상태 폴링**으로 노출한다(예: "추출 중… 45/576").

### R2. 검수 화면 (F2)

- 파싱 결과를 표로 보여주고, 행 단위 **수정·삭제·추가**를 지원한다.
- 파싱이 애매/실패한 행은 별도 표시해 사용자가 우선 확인하도록 한다. (Flash 단일 모델 선택으로 파싱 오류 가능성이 있으므로 이 강조 표시는 필수)
- 사용자가 **"이 단어장으로 확정"**을 클릭하면, 그 시점에 확정된 단어 목록으로 R3(생성)을 시작한다. **확정 전에는 AI 생성을 호출하지 않는다.**

### R3. AI 유의어·예문 생성 (F3)

- **타이밍: 검수 확정 후** 배치 생성.
- 확정된 단어를 **20개씩 묶어** 1회 호출한다(**Pass 2**, JSON 배열 반환).
- 단어당 생성 내용: **유의어 1~3개** + **예문 2개**(각 예문은 `jp` / `reading` / `ko`).
- 생성 결과는 `Word.synonyms`, `Word.examples`에 **영구 저장**하고, 동일 단어를 재호출하지 않는다.
- 생성 진행률은 상태 폴링으로 노출한다(예: "생성 중… 320/576").
- **청크 실패 시 해당 청크만 재시도(최대 3회)**. 전체 재실행 금지.

**생성 결과 JSON 스키마(예시):**
```json
{
  "kanji": "暖まる",
  "reading": "あたたまる",
  "meaning_ko": "따뜻해지다, 훈훈해지다",
  "synonyms": ["温まる", "ぬくもる"],
  "examples": [
    { "jp": "スープを飲んで体が暖まった。", "reading": "スープをのんでからだがあたたまった。", "ko": "수프를 마시고 몸이 따뜻해졌다." }
  ]
}
```

### R4. 단어장 뷰 (F4)

- 단어 카드 리스트로 `漢字 / 후리가나 / 뜻 / 유의어 / 예문`을 표시한다.
- 각 단어에 SRS 레벨 뱃지(0~5)를 표시한다.
- 검색 및 필터: 레벨별, 미학습/복습필요.

### R5. 퀴즈 & 복습 (F5)

- 유형: **4지선다, 양방향.**
- **정방향**: 문제 = `漢字 + 후리가나`(kana 단어는 후리가나만), 보기 = 정답 뜻 + 같은 단어장 내 랜덤 오답 뜻 3개.
- **역방향**: 문제 = `한국어 뜻`, 보기 = 정답 단어 + 랜덤 오답 단어 3개.
- **세션 크기: 기본 20문제.** 시작 화면에서 10 / 20 / 전체(출제 대상 전부) 중 선택할 수 있다. *(개정 2026-07-06)*
- **출제 우선순위**: ① 오늘 복습 대상(`next_review_date <= today`) → ② 미학습 단어.
- 정답/오답 결과는 R6 SRS 로직으로 레벨·복습일을 갱신한다.
- 세션 종료 시 정답률·소요시간·틀린 단어를 요약한다.

**답변 인터랙션 (개정 2026-07-06 — 사용성 피드백):**
- 답을 선택하면 **즉시 다음 문제로 넘기지 않는다.** 선택한 보기와 정답을 정/오 색상으로 표시하는 **피드백 단계**를 거친 뒤, "다음" 버튼으로 진행한다. (기존 즉시 전환은 화면이 깜빡이고 문제를 건너뛴 듯 보이는 문제가 있었음)
- **힌트 보기**: 문제당 1회, 정답 단어의 예문(일본어 문장, 한국어 번역은 숨김)을 힌트로 보여준다. 예문이 없는 단어는 유의어를 보여준다(둘 다 없으면 버튼 비활성).
- **정답 보기**: 모르는 문제는 정답을 즉시 공개할 수 있다. 이 경우 **오답으로 기록**하고 피드백 단계로 넘어간다.

### R6. 경량 SRS — Leitner 박스 (F6)

- 단어별 숙련도 레벨 0~5.
- 복습 간격표:

| 레벨 | 다음 복습까지 |
|------|---------------|
| 0 | 당일(즉시 재출제) |
| 1 | +1일 |
| 2 | +3일 |
| 3 | +7일 |
| 4 | +14일 |
| 5 | +30일 |

- **정답**: `level = min(level+1, 5)`, `next_review_date = today + interval[level]`.
- **오답**: `level = 0`, `next_review_date = today` (레벨 0 정의와 일치, 즉시 재출제).
- 매 시도마다 `last_reviewed_at`, `correct_count` / `wrong_count`를 기록한다.

### R7. 진도 대시보드 (F7)

- **진도율 = (레벨 4 이상 단어 수) / (전체 단어 수) × 100.**
- 오늘 복습할 단어 수, 미학습 단어 수 표시.
- 레벨별 분포(0~5) 막대그래프.
- 단어장별 진도.

### R8. AI 파이프라인 (2-Pass 유지)

- 파싱(Pass 1)과 생성(Pass 2)을 분리해 오류 전파를 막는다.
- 모든 호출은 모델 `gemini-2.5-flash`, API 키는 `GEMINI_API_KEY` 환경변수로 주입한다.
- 시스템 프롬프트에 "JSON 외 텍스트·마크다운 금지"를 명시한다.
- 응답은 ` ```json ` 펜스 제거 후 `JSON.parse`하고, 실패 시 재시도한다.

### R9. 화면 구성 (MVP)

1. 홈/대시보드 — 진도율, 오늘 복습 수, 단어장 목록, 레벨 분포
2. 업로드 — PDF 드롭존 → 파싱 진행바(폴링)
3. 검수 — 파싱 결과 표(수정/삭제/추가) → 확정 → 생성 진행바(폴링)
4. 단어장 상세 — 단어 카드 리스트
5. 복습/퀴즈 — 4지선다 진행 화면(20문제)
6. 결과 — 세션 요약(정답률·틀린 단어)

### R10. 공통 UI/UX (개정 2026-07-06 — 사용성 피드백)

MVP 사용 결과 화면 간 이동 수단이 없어 흐름이 끊기는 문제가 확인됨. 다음을 필수로 한다.

- **글로벌 헤더**: 모든 페이지 상단에 고정. 홈 링크(로고/서비스명)와 **"새 단어장"(업로드) 버튼**을 항상 노출한다. 단어장 생성이 빈 상태에서만 가능하던 문제를 해소한다.
- **목록 → 상세 내비게이션**: 홈 단어장 목록의 각 항목은 단어장 상세(`/vocab-sets/:id`)로 이동하는 링크다.
- **로딩 피드백**: 페이지 전환과 비동기 작업(업로드, 확정 등) 중에는 로딩 인디케이터(스피너 또는 스켈레톤)를 표시한다. 잡 진행률(파싱/생성)은 기존 폴링 진행바를 유지한다.
- **디자인 방향 — 모던**: 색·간격·타이포·버튼 위계(primary/secondary)·카드 스타일을 `docs/design-system.md`로 정의하고 전 화면에 일관 적용한다. 기조: 여백을 살린 카드 레이아웃, 라운드·그림자 절제, 포인트 컬러 1개(blue 계열), 상태 색(정답/오답/실패)은 용도 외 재사용 금지.

## 경계 조건 및 에러 처리

### 업로드/파싱
- 파일이 PDF가 아니거나 10MB 초과 → 업로드 거부, 사용자에게 사유 표시.
- unpdf 텍스트 추출 결과가 비어 있음(스캔 이미지 PDF 등) → "텍스트를 추출할 수 없습니다" 안내, 파싱 진행 중단.
- Pass 1 응답이 JSON 파싱 실패 → 해당 청크 재시도(최대 3회), 최종 실패 시 그 청크를 "파싱 실패"로 표시하고 검수 화면에서 사용자가 수동 처리.
- 파싱 결과 0건 → 빈 검수 화면 대신 "인식된 단어가 없습니다" 안내.

### 검수
- 검수에서 모든 행을 삭제하고 확정 시도 → 확정 불가(최소 1개 필요) 안내.
- 중복 단어(같은 kanji+reading) → 경고 표시하되 사용자가 유지/삭제 선택.
- `reading`은 필수. 비어 있는 행은 확정 전 채우도록 요구(kana 단어는 kanji=null 허용).

### AI 생성
- Pass 2 청크 실패 → 해당 청크만 재시도(최대 3회). 최종 실패한 단어는 `synonyms`/`examples`가 비어도 저장은 진행하고, 단어장 뷰에서 "생성 실패, 재시도" 표시.
- 생성 도중 사용자가 이탈 → job 상태는 서버에 남아, 재방문 시 진행률을 폴링으로 이어서 표시.

### 퀴즈
- 오답 보기 3개를 뽑을 만큼 단어가 충분하지 않은 단어장(4개 미만) → 퀴즈 시작 불가, "단어가 4개 이상이어야 합니다" 안내.
- 출제 대상(복습+미학습)이 20개 미만 → 있는 만큼만 세션 구성.
- 오늘 복습 대상과 미학습이 모두 0 → "오늘 복습할 단어가 없습니다" 안내.

### 동시성
- 1인 개인용·무인증이므로 동시 편집 충돌은 범위 밖. 단, 생성 job은 단어장당 1개만 동시 실행(중복 실행 방지).

## 용어 정의 (Ubiquitous Language)

코드·문서·대화에서 아래 용어를 일관되게 사용한다.

| 용어 | 정의 |
|------|------|
| VocabSet(단어장) | 업로드한 PDF 한 개에서 만들어진 단어 묶음 |
| Word(단어) | 파싱 결과 + AI 생성 결과를 통합한 단일 단어 레코드 |
| Parsing(파싱, Pass 1) | PDF 텍스트에서 `{kanji, reading, meaning_ko}` 구조를 추출하는 AI 단계 |
| Enrichment(생성, Pass 2) | 확정된 단어에 유의어·예문을 붙이는 AI 단계 |
| Review(검수) | 파싱 결과를 사용자가 수정·삭제·추가 후 확정하는 화면/행위 |
| 확정(Confirm) | 검수 완료 후 단어장을 잠그고 Enrichment를 트리거하는 행위 |
| Reading(후리가나) | 단어의 읽기(かな). 소스값 우선, 없을 때만 AI 보완 |
| SRS Level(레벨) | 단어별 숙련도 0~5 (Leitner 박스) |
| next_review_date(다음 복습일) | 이 단어가 다시 출제되는 날짜 |
| 정방향(word_to_meaning) | 단어→뜻 방향 4지선다 |
| 역방향(meaning_to_word) | 뜻→단어 방향 4지선다 |
| 미학습 단어 | SRS 기록이 없거나 한 번도 출제되지 않은 단어 |
| 복습 대상 | `next_review_date <= today`인 단어 |
| 진도율 | (레벨 4 이상 단어 수) / (전체 단어 수) × 100 |
| Enrichment Job(생성 잡) | 단어장 하나에 대한 Pass 2 배치 실행 단위. 진행률은 폴링으로 조회 |

## 이번 MVP에서 제외 (spec-original 8장 유지)

주관식/빈칸 퀴즈, AI 생성물 인라인 수정, TTS, 반의어·JLPT·품사, 멀티유저(로그인), 상세 통계 그래프, 학습 스트릭. (인라인 수정은 P1로 계획됨)
